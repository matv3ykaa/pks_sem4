# Анализатор сетевых подключений

Практическое задание по дисциплине «Программирование корпоративных систем».  
Реализовано на **Avalonia UI** (кросс-платформенный аналог WPF) — работает на macOS (Apple Silicon), Windows и Linux.

## Требования

- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)

## Запуск

```bash
cd NetworkAnalyzer
dotnet run
```

Первый запуск выполнит `dotnet restore` автоматически (скачает пакеты Avalonia ~30 МБ).

## Сборка в бинарник

```bash
# macOS ARM64
dotnet publish -c Release -r osx-arm64 --self-contained

# Windows x64
dotnet publish -c Release -r win-x64 --self-contained
```

## Функциональность

### Сетевые интерфейсы
- Список всех интерфейсов с иконкой состояния (● активен / ○ неактивен)
- Фильтрация по имени/описанию
- Для каждого интерфейса: IP-адрес (IPv4 и IPv6), маска подсети, MAC-адрес, состояние, тип, скорость
- Статистика: всего / активных / неактивных
- Кнопка обновления

### Анализ URL / URI
- Парсинг через `System.Uri`: схема, хост, порт, путь, параметры запроса, фрагмент
- Поддерживаемые схемы: http, https, ftp, ftps
- Автодобавление `https://` при отсутствии схемы
- Проверка доступности хоста (Ping, таймаут 3 сек)
- DNS-разрешение (таймаут 5 сек)
- Определение типа адреса: loopback, приватный (RFC 1918), link-local, публичный, mDNS, доменный и т.д.
- История проверенных URL (до 100 записей), клик по записи подставляет URL в поле ввода
- Enter в поле URL запускает анализ

### Обработка ошибок
Обрабатываются: `NetworkInformationException`, `SocketException` (все коды), `PingException`, `TimeoutException`, `ArgumentException`, `InvalidOperationException`, управляющие символы в URL, превышение длины URL (2048), длины хоста (253), длины фильтра (100), переполнение буфера результатов (50 000 символов).
